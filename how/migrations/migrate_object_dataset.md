# Migration: Dataset — pre-LSU → LSU-1.0

## Summary

Upgrades vault datasets to full aDNA compliance by normalizing `dataset_class`, enriching FAIR metadata, adding federation stubs, and generating companion `.dataset.yaml` files for federation-critical datasets. Scope is limited to ~5-8 main datasets per FA Decision D3; targets may get companions if bioinformatics-relevant.

**Gaps addressed**: G-004 (zero `.dataset.yaml` companions exist for 22 datasets), G-013 (federation metadata absent).

## Scope

- **Object type**: dataset (includes both main datasets and targets)
- **Discovery**: Files matching `type: dataset` in frontmatter
- **Location**: `what/datasets/*.md` (main) and `what/datasets/targets/*.md` (bio targets)
- **Version gate**: Skip if `_migration_version: "lsu-1.0"` already present
- **Companion scope per D3**: ~5 main datasets + select targets that are bioinformatics-relevant. Not all 22.
- **Companion generation criteria**: A dataset gets a `.dataset.yaml` if:
  - It is a main dataset (not a target), OR
  - It is a target referenced in an active bio lattice (e.g., `lattice_sjogrens_binder_design`)

### `dataset_class` Canonical Values

| Current | Canonical |
|---------|-----------|
| `reference`, `ref` | `reference` |
| `training`, `train` | `training` |
| `target` | `target` |
| `benchmark`, `eval` | `benchmark` |
| `derived`, `processed` | `derived` |

If `dataset_class` is absent, determine from context:
- Files in `what/datasets/targets/` → `target`
- MSA databases, PDB structures → `reference`
- Training sets → `training`
- Derived/processed outputs → `derived`

### `fair.keywords` Guidance

Read the dataset's description or `## Overview` section. Extract 1-3 keywords:
- `dataset_msa_databases` → `["protein", "multiple-sequence-alignment", "bioinformatics"]`
- `dataset_pdb_structures` → `["protein", "crystal-structure", "pdb"]`
- `target_la_p05455` → `["sjogrens", "autoantigen", "la-protein"]`

### `fair.license` Guidance

- Public databases (PDB, UniProt-sourced): `"CC-BY-4.0"`
- Internal/generated datasets: `"Apache-2.0"` (vault default)
- External datasets: Check source license. If unknown, use `"CC-BY-4.0"` with `# TODO: verify license`
- Targets (disease graphs): `"Apache-2.0"`

## Steps

### Step 1: Read Current State

Read the dataset file. Parse frontmatter. Record current fields.

### Step 2: Normalize `dataset_class`

If `dataset_class` uses a non-canonical value, replace with the canonical equivalent. If absent, add based on context (see mapping above).

### Step 3: Add/Enrich FAIR Block

If no `fair:` block exists, add it:

```yaml
fair:
  keywords: ["keyword1", "keyword2"]
  license: "CC-BY-4.0"
```

If a `fair:` block exists, add missing fields without overwriting existing values.

### Step 4: Add Federation Stub

If no `federation:` block exists, add it:

```yaml
federation:
  discoverable: true
```

### Step 5: Generate Companion `.dataset.yaml` (if in scope)

**Skip this step if**: Dataset does not meet the companion generation criteria (see Scope).

Follow the 5-step companion generation process:

#### 5a. Extract

Pull from the `.md` frontmatter:
- `name` (from filename, e.g., `dataset_msa_databases` → `msa_databases`)
- `dataset_class` (from Step 2)
- `version` (if present)
- Storage location hints from body content

#### 5b. Template

```yaml
# <dataset_name>.dataset.yaml
# Generated by migrate_object_dataset (lsu-1.0)
name: "<dataset_name>"
dataset_class: "<dataset_class>"
version: "<version or '0.1.0'>"

storage:
  provider: "<provider or null>"
  location: "<path or URI or null>"
  format: "<format or null>"

lineage:
  source: "<source description or null>"
  processing: "<processing steps or null>"

fair:
  keywords: []  # Copied from .md
  license: "<license>"

federation:
  discoverable: true
```

#### 5c. Judge

Agent determines these fields by reading the dataset's body content:
- **`storage.provider`**: Look for cloud provider mentions (S3, GCS, Azure, local). Common: `"local"`, `"s3"`, `"gcs"`
- **`storage.location`**: Look for file paths, URIs, or mount points
- **`storage.format`**: Look for file format mentions (FASTA, PDB, CSV, parquet, JSON)
- **`lineage.source`**: Where the data came from (e.g., "UniProt release 2024_05", "PDB mirror")
- **`lineage.processing`**: Any processing steps described

If a field cannot be determined, set to `null`. Do not guess.

#### 5d. Validate

Before writing, verify:
- YAML parses without error
- `name` matches the dataset name
- `federation.discoverable` is `true`

#### 5e. Write

Save alongside the `.md` file:
- Main datasets: `what/datasets/<dataset_name>.dataset.yaml`
- Targets: `what/datasets/targets/<target_name>.dataset.yaml`

**Naming convention**: `<name>.dataset.yaml` alongside `<name>.md`.

### Step 6: Add Migration Version

Add as the last frontmatter field in the `.md` file:

```yaml
_migration_version: "lsu-1.0"
```

### Step 7: Verify

Re-read the `.md` file and companion (if generated). Confirm YAML parses, all target fields present, body unchanged.

## Rollback

- **L1** (single file): `git checkout -- <md_path>` and `rm <yaml_path>` (if companion was generated)
- **L2** (field-level): Revert specific frontmatter fields. Remove added `fair`, `federation`, `_migration_version` fields.
- **L3** (batch): `git reset --hard <pre-migration-tag>` (e.g., `pre-lsu-m08`)
