## Extended findings (3 categories × 4 findings each = 12)

### Category 1 — Cross-skill primitive composition pattern graduation at 3/3 (PRIMARY substrate; × 4)

1. **TRIPLE DELEGATION at T8d is the strongest single-artifact demonstration of the pattern in v8.** T8d skill (`skill_obsidian_agent_inspect.md`) delegates to THREE separate skills within a single skill body — T6 (`skill_obsidian_integration_test.md`; depth 1 = direct delegation for binary-presence verification) + T7 (`skill_verification_handoff.md`; depth 1 = dispatch consultation) + M3.2 skill (`skill_obsidian_canonicalize.md`; depth 3 = reached via T6 chain → setup.sh `--verify` → Obsidian state). Three independent delegation paths within one skill body. The chain depth-analysis (T8d → T6 → M3.2 skill → setup.sh = 4 layers via T6; T8d → T7 = 1 layer for dispatch; T8d → T6 = 2 layers for operator_side fallback in degradation cascade) stays within M3.3's operator-traceable ≤ 5-6 layer ceiling. No layer reimplements another's responsibility. This is the clearest test the pattern has faced, and it passed without scope creep.

2. **The pattern's 3 ratified instances trace an evolution of complexity that ratifies its generality.** Instance 1 (M3.3 T6 → M3.2 skill) was a 4-layer linear chain — the simplest delegation shape. Instance 2 (M3.4 T7 → T6 skill) inherited the linear shape but added a layer (now 5 layers max via T7 → T6 → M3.2 skill → setup.sh → Obsidian state) — still operator-traceable. Instance 3 (M3.4 T8d → T6 + T7 + M3.2 skill) is the first **branching** delegation graph (3 independent paths from a single source skill). The pattern generalizes from linear chains to delegation graphs without requiring methodology amendment — the discipline's core (cite + invoke; document depth; name responsibility boundary) scales to graphs by simply repeating the documentation per path. **Graduation criterion satisfied** per `m21_obj4` rubric (≥ 3 use instances); `skill_cross_skill_primitive_composition.md` skill file authoring deferred to follow-up per D-GRAD operator choice (backlog idea filed at `how/backlog/idea_pattern_graduation_skill_authoring.md`).

3. **The pattern's degradation cascade is a 2nd-order discipline that emerged at M3.4 T8d.** T8d's `--mode {agent_driven, hybrid, operator_side}` slot with graceful degradation (if MCP unreachable → degrade to `hybrid`; if REST API key invalid → degrade to `operator_side`) is the first explicit degradation-cascade contract in M3.x cohort. Cross-skill primitive composition becomes safety-critical when one delegation path may become unavailable at runtime — the cascade contract is how a delegating skill survives downstream skill failures gracefully. The pattern's next refinement (post-graduation) likely codifies degradation-cascade as a 2nd-order discipline: any skill with ≥ 2 delegation paths SHOULD document a degradation cascade per path. Backlog idea seeded inline.
