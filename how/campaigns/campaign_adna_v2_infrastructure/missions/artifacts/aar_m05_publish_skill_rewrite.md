### Surprises and friction

1. **Skill-path mismatch defect class (post-M03-flatten path-form drift)**: 4 v7.0 skill files (`skill_deploy.md`, `skill_vault_publish.md`, `skill_publish_tarball.md`, plus hook header) referenced `.adna/how/standard/hooks/pre-push-sanitize.sh` (pre-M03-flatten path, when vaults had child `.adna/` template subdir). Post-flatten reality (per [[../../../skills/skill_project_fork.md|skill_project_fork.md]] line 84-90 + [[../../../../CLAUDE.md|aDNA.aDNA CLAUDE.md]] Standing Rule 1): vaults are SIBLINGS of `.adna/` and contain COPY of template content as vault-local — no child `.adna/`. Rewrite spec line 312 explicitly anticipated BOTH paths ("template-shipped" vs "inherits/customizes locally") but S1 implementation only checked one. Fix: change all 4 files to check `how/standard/hooks/...` FIRST (canonical post-flatten), fallback `.adna/how/standard/hooks/...` (legacy). Hook header also updated to name both source locations. **Lesson**: when a spec mentions multiple paths, the implementation must implement both (or assert that one is canonical-only); reviewer should explicitly enumerate path variants checked. **Defect class**: post-restructuring path-form drift is a regression risk for any future template-restructuring mission (M11 / v8 P1 if structure changes).
2. **R2 quote-class regex defect (POSIX-vs-PCRE escape interpretation)**: all 7 patterns in `secret_patterns` used `[\x27\x22]?` to mean "optional single or double quote". POSIX `grep -E` does NOT interpret `\xNN` as hex escapes — that's PCRE territory. With legacy `'...'` quoting, the patterns contained 8 LITERAL characters `\`, `x`, `2`, `7`, `\`, `x`, `2`, `2` — never matching `'` or `"`, silently missing every QUOTED secret in production push-time R2 scans. Surfaced when self-test fixture (quoted AWS-style key) failed to match. Fix: switch all 7 patterns from `'...'` to bash ANSI-C quoting `$'...'` (which DOES expand `\x27`/`\x22` to `'`/`"`); preserve `\-` via `\\-` to keep regex semantics intact. **Lesson**: even reviewed hook code can ship with silent regex bugs that only surface against carefully-crafted fixtures. **Defect class**: regex escape-interpretation drift between shell quoting modes (literal `'...'` vs ANSI-C `$'...'`) is a recurring trap — particularly when migrating from PCRE-aware tools (grep -P, awk, sed -E in some implementations) to strict POSIX. Worth a Standing Order note: when authoring hook regex in bash, prefer ANSI-C quoting for any escape sequence.
3. **gh token `delete_repo` scope gap**: gh CLI authenticated with `repo` scope at S2; `delete_repo` not in scope. Scratch repo `ScienceStanley/m05-test` cleanup deferred to operator post-session (added to STATE.md Pending Manual Actions). **Friction-level**: minor. **Pattern**: verification sessions that create throwaway remote resources should pre-flight check gh scope, OR plan for operator-side cleanup as a documented item. M06 v7.0 tag verification (if planned) should pre-flight verify `delete_repo` scope is granted before creating a scratch tag.
4. **R5 + R7 fixture coverage deferred**: R5 (`confidential: true` / `private: true` frontmatter trigger) and R7 (operator-defined deny list at `.adna/sanitize_deny.txt`) are gated in production push-time scans but not exercised by the 8 self-test fixtures. Spec §5 explicitly defers these to first operator use. **Friction-level**: minor — no production gap; just less self-test coverage. **Risk**: future hook revisions could regress R5 or R7 without detection. **Mitigation candidate**: M07 absorb by authoring 2 more fixtures (`dirty/confidential_doc.md` for R5 + a synthetic `dirty/in_deny_list.md` paired with a fixture-local deny file).
