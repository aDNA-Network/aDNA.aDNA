# UX Heuristic Audit — 2026-04-23

## Why This Audit Exists

Phase 7's existing ranker answers "does this work for my persona?" It does not answer "is this well-made, and would a stranger feel welcomed?" Delight has been flat at 4.0 across all 5 adopters through D1 + D2, and Lighthouse saturates at 100/100/100/100 — both signals that automated gates are no longer finding what's missing. This audit walks 4 newcomer paths with a cold-reader lens and routes each finding to the decadal that owns it.

**Audit constraint**: evidence-only. Every finding cites a file path, a line number, or a measured number. No speculation.

**Self-reference**: this artifact is the vault reviewing the site that publishes the vault. The finding pattern (severity-ranked, decadal-routed, dual-audience) is itself an aDNA governance move — critique flows into the Campaign → Mission → Objective hierarchy via the priority queue below.

## Findings — Severity-Ranked

Severity grades: **Critical** (blocks newcomer), **High** (significantly degrades newcomer), **Medium** (visible to attentive newcomer), **Low** (polish).

### F-01 [HIGH] · Homepage hero-lead is jargon-dense, fails plain-language test

**Location**: `site/src/pages/index.astro:89`

**Evidence**: The hero-lead reads *"aDNA is an integrated standard for knowledge graph driven context engineering."* This sentence contains four simultaneous abstractions (*integrated standard*, *knowledge graph*, *driven*, *context engineering*). A 14-year-old, a product manager, or a curious CEO cannot parse it unaided.

Compare `/learn/what-is-adna:18` (the sub-page the hero CTA links to): *"aDNA (Agentic DNA) is an open standard for organizing project knowledge so that both humans and AI agents can navigate it. Think of it as a genome for your project — a structured encoding of what your project knows, who is involved, and how work gets done."* — plainer, grounded in a metaphor, dual-audience.

The homepage subtitle (`index.astro:90`) recovers: *"A shared folder layout and metadata spec so AI agents land oriented — no re-learning your project every time."* — but the damage is done at the lead.

**Violation**: Campaign standing rule #6 (plain-language opening: "1–3 sentences a 14-year-old could follow; technical depth comes after the on-ramp").

**Impact**: this is the single highest-leverage newcomer barrier. The tagline is the first non-image thing the reader encounters and it signals "this is not for you" to non-specialists.

**Route**: D4 (Visual Identity & First-Contact) — assign to a cycle 31–33 target. Recommend replacing the lead with the `/learn/what-is-adna` framing and promoting the current tagline to a secondary "for specialists" line below.

### F-02 [HIGH] · The canonical "What is aDNA?" page under-delivers for the weight placed on it

**Location**: `site/src/pages/learn/what-is-adna.astro:12–37`

**Evidence**: The page is ~130 author words in 4 short sections (intro, problem, solution-as-3-bullets, explore-further). The homepage hero CTA reads "What is aDNA?" and sends the reader here; the page gives a textbook-opener paragraph and stops. There is:
- No diagram
- No worked example
- No self-reference to this vault as a working instance
- No concrete file paths (CLAUDE.md appears once, no path)
- No acknowledged trade-offs
- Only 3 outbound wikilinks (`/learn/concepts/triad`, `/get-started`, `/learn/tutorials/first-claude-md`)

**Violation**: Campaign standing rule #8 (self-reference mandatory) — absent. Rule #10 (≥2 wikilinks) — barely satisfied.

**Impact**: Newcomer lands here, reads 130 words, learns "aDNA is a standard with three parts," and must decide whether to invest in Triad / Get-Started / Tutorial without enough context to choose well.

**Route**: M27 O7 or new D3-residual objective — expand to ~600–800 words (within concept file token budget, campaign rule #4). Add: (a) a worked "before/after" example showing an agent session WITH vs. WITHOUT aDNA, (b) self-reference pointer to `aDNA.aDNA/CLAUDE.md` as a live instance, (c) a 3-sentence "what it's not" section to set scope expectations, (d) honest trade-offs per voice guide.

### F-03 [HIGH] · The "What / Why / How" crispness test fails on "Why" and "How"

**Scorecard** (answers a cold-reader can give after scanning homepage + `/learn/what-is-adna` for 60 seconds):

| Axis | Can stranger answer after 60s? | Evidence |
|------|-------------------------------|----------|
| **What** is aDNA? | YES (weak) | "An open standard for organizing project knowledge." Homepage + explainer converge. |
| **Why** does aDNA exist? | NO | The pain point "agents waste tokens or miss context" appears once on `/learn/what-is-adna:21` as an abstract claim. Homepage never states the problem directly — it asserts benefits (*Agent-native*, *Human-readable*, *Composable*) without establishing the gap those benefits close. |
| **How** does it work? | NO | Homepage "How it Works" section lists Structure → Orient → Execute (`index.astro:10–29`) — all three phrases are so high-level they are tautological. A newcomer learns that "aDNA organizes things" but not the working mechanism (triad, governance files, typed entities, convergence hierarchy). The real "how" is in `/learn/concepts/triad` — **two clicks away**. |

**Impact**: Newcomers leave the top two pages with a brand impression ("an organizing standard for AI agents") but no working mental model. This drags the Comprehension and Relevance ranker dimensions and is the most likely cause of Delight being flat at 4.0 (can't delight in something you don't understand).

**Route**: D4 (Visual Identity & First-Contact) owns the homepage rewrite. A new sub-theme **Explanatory Scaffolding** should be absorbed into D4 or become a D9 sub-theme (Persona-Driven + Narrative Coherence). Recommended edits:
1. Homepage "How it Works" becomes **"The Problem → The Shape → The Win"** — a 3-step explanatory arc (pain state → structural fix → agent outcome) with one concrete file path per step.
2. The current "Structure / Orient / Execute" trio demotes to a sidebar or moves into `/learn/concepts/triad`.

### F-05 [MEDIUM] · 12+ decorative emoji used as icons — generic-AI-aesthetic tell

**Location**: `site/src/pages/index.astro` — 🗂🧭🚀 (steps), 👩‍💻🏢🎓⚡🔬 (personas), 🎯📐🧩🏛️ (context cards)

**Evidence**: 12 emoji used decoratively as section-icons across four sections. This is a strong aesthetic signature of generative-AI-produced landing pages (ChatGPT marketing sites, Cursor-generated portfolios). It works against the vault's "anti-jargon, warm-and-precise" voice and against the brand line in the Identity section of `CLAUDE.md` that emphasizes self-referential rigor.

**Impact**: Delight dimension ranker captures this only weakly (persona review catches "does this feel right to me" but often forgives familiar patterns). A Design Critic reviewer would flag every instance.

**Route**: D4 (Visual Identity & First-Contact). Options:
- Custom SVG icon set tuned to the aDNA palette (preferred — distinctiveness)
- Typographic section markers (numerals, small-caps labels) — minimalist route
- Keep emoji but restrict to 1 per section max and pair with a stronger typographic anchor

### F-07 [MEDIUM] · Homepage is structurally heavy — 7 sections competing for attention

**Location**: `site/src/pages/index.astro` — Hero, Features (3), How it Works (3 steps + placeholders), Operational Ontology (3-column grid, 14 items), Context Engineering (4 cards), Who Uses aDNA (5 persona cards), The Standard (stats + CTA).

**Evidence**: 7 top-level sections, 19 sub-cells (3+3+3+4+5+1 across the content sections), 738 visible words. Information density per section is low (avg ~100 words/section) but section count is high. Classic "kitchen sink" landing pattern.

**Impact**: First scan costs the reader ~45 seconds to map the structure before any content lands. Each section competes with the others for the one slot in working memory. A newcomer retains the tagline and maybe one of the 19 sub-cells.

**Route**: D4 (Visual Identity & First-Contact) — consider collapsing to 4 sections:
1. **Hero + Problem** (replaces current Hero + Features)
2. **How it Works** (expanded, with real mechanism)
3. **Who Uses aDNA** (keep — personas are high-leverage for self-selection)
4. **The Standard** (keep — trust signals)

Demote Ontology + Context Engineering cards to `/learn/` index pages where a reader who has committed to learning will find them.

**Location**: `site/src/components/common/Header.astro` — Learn, Use Cases, Patterns, Glossary, Community, How, Reference

### F-10 [LOW] · Trust signals buried under "The Standard" (last section)

## Answers to the Three Headline Questions

### Are pages too big?

**No — and the more acute problem is that the explainer page is too small.** Tutorial 1116 words and homepage 738 words are both well within campaign token budgets (rule #4). The homepage feels heavy not from word count but from **section count** (F-07). The explainer page (`/learn/what-is-adna`) is ~130 author words of real content — under-delivers on the CTA's promise (F-02).

### Is the site accessible to new users?

**Automated a11y: perfect. Newcomer-cognitive a11y: C+.** WCAG 2.1 AA passes site-wide (axe-core zero violations, 20+ pages tested). But newcomer-cognitive accessibility — the ability of someone with no aDNA context to build a working mental model — is degraded by:
- Jargon-dense hero tagline (F-01)
- Missing pain-state framing (F-03)
- Abstract "How it Works" that doesn't show the mechanism (F-03)
- Assumed Claude Code knowledge (F-08)
- Generic-AI visual signals that subtly signal "not for serious readers" (F-05)

### Does the site crisply explain what aDNA is / what it's for / how it works?

**What**: yes (weakly). **Why**: no. **How**: no.

See F-03 scorecard. The shortest path to "crisp" is:
1. Rewrite the homepage hero-lead in plain language (F-01).
2. Expand `/learn/what-is-adna` to a full concept file with before/after example + self-reference + trade-offs (F-02).
3. Replace the current homepage "How it Works" with a Problem → Shape → Win arc that names the mechanism (F-03).

If only those three land, the audit's headline concerns resolve without touching any other finding.
