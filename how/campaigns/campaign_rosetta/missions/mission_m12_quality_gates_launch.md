## Issues Fixed During Gate Execution

1. **G2 type errors (4)**: `guides` and `reference` collection schemas missing `draft` field. Added `draft: z.boolean().default(false)` to both schemas in `content.config.ts`.

2. **G4 color contrast violation**: `.difficulty-badge.beginner` had insufficient contrast (2.46:1 vs 4.5:1 required). The `--color-success` green on 15% green background was too light. Fixed by using darker badge text colors: `hsl(142 72% 28%)` for beginner (5.4:1 contrast).

3. **G9 horizontal overflow at 320px**: Tables in MDX content overflowed the viewport on narrow screens. Fixed by adding `display: block; overflow-x: auto` to `.prose table` in global.css, and `overflow-x: hidden` on `.doc-content` in DocumentationLayout.

## Known Trade-offs

1. **`script-src 'unsafe-inline'`** in CSP — required for dark mode init inline script. Defer nonce/hash migration to M19.
2. **No OG image** — `og-default.png` doesn't exist. OG tags present but image URL returns 404. Defer image creation to M19.
3. **Vercel project name is `site`** — auto-created. Custom domain (`adna.dev`) to be configured separately.

## AAR

- **Worked**: Lean Playwright gate suite was the right call — 30 tests covering G4/G6/G8/G9 in 20 seconds. Lighthouse CLI scores were excellent (100/95/100/100) for a static site with zero client JS.
- **Didn't**: Initial G2 run exposed 4 type errors from missing `draft` field on `guides` and `reference` schemas. Should have caught this in M10 schema design.
- **Finding**: `vercel.json` headers only work with standard (non-prebuilt) deploys. Prebuilt deploys use `.vercel/output/config.json` which gets regenerated by the Astro adapter.
- **Change**: Badge text colors now use WCAG-compliant darker shades rather than raw semantic color tokens. Tables in prose have horizontal scroll for narrow viewports.
- **Follow-up**: M19 (Website v2) — OG image, CSP nonce migration, custom domain configuration, Phase 4-5 content sync.
