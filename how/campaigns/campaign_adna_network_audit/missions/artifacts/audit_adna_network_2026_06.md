## 1. Executive summary — top findings (ranked by impact)

1. **Vault-detail pages ship raw internal campaign jargon as public meta/OG descriptions — /vaults/III.aDNA/ description reads 'Campaign C DG-C closed 2026-05-20; Campaign D CHARTERED. 6 consumer wrappers live.' (verified verbatim in description AND og:description) [local|live].**  
   → _Critical credibility self-own across the largest page class (~37 pages); search snippets and link-preview cards make the public standard read as an internal scratchpad._
2. **Naming-discipline break: the vault registry is labeled 'The Lattice' in global nav, page title, H1, and back-links — conflating the forward-facing aDNA brand's vault collection with the underlying Lattice Protocol IP (verified Header.astro:6, vaults.json title/H1) [local|live].**  
   → _Critical. Blurs the one distinction the project treats as load-bearing; same objects are also called 'the network' / 'vaults' / 'registry' elsewhere, so visitors never form a stable model._
3. **Two broken GitHub repo references at the conversion peak: get-started.astro:51 'See it live' → github.com/LatticeProtocol/aDNA-aDNA is a 404, and every sitewide GitHub link (Header:37, Footer:12, HomeHero:32, get-started clone:29) points to Agentic-DNA which 301-redirects, not the canonical /aDNA [local|live].**  
   → _Critical. A 404 at the highest-intent clone moment plus a redirecting/old-named canonical source repo — direct credibility hit to the exact technical audience the site must convert._
4. **The standard contradicts its own version number: home + spec frontmatter say v2.2, but specification.mdx:1479 closes 'End of aDNA Universal Standard v2.0' and open-standard.mdx + 5 comparison docs + design-rationale cite v2.1 (verified) [local|live].**  
   → _High. A spec whose entire value is normative authority disagrees with itself on its version, reading as unmaintained to spec-evaluators._
5. **Conversion funnel is action-poor: no styled 'Get Started' CTA exists in the header, home hero, or on any audience/community/vault-detail page (only a footer text link); home's four styled CTAs all route to reading or browsing, never building (verified home ctas[]) [local|live].**  
   → _High. Every high-intent 'how do I start?' visitor must scroll-hunt the footer; targeted pages dead-end with only theme/menu toggles registering as CTAs._
6. **Deploy gap: the strongest message surface /commons ('A commons, not a catalog', axe-clean) is built but (a) fully orphaned — zero inbound links anywhere in src/, and (b) not on live; live also advertises a stale 40-vault count vs HEAD's 37; /commons publication is correctly blocked by the ADR-010 Wilhelm-Foundation embargo (named individuals in subnetworks.json:57).**  
   → _Medium-High. The best 'why it matters' asset is invisible to real visitors and unreachable even locally; deploying as-is would also publish un-cleared partner attributions and the 404/301 repo breaks._
