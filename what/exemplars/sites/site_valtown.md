# site_valtown — Val Town

> **The young-network exemplar** — the most realistic comparable for aDNA *today*. Shows how a small/young registry+community feels **alive without vanity metrics**: trust comes from *who* uses it, not *how many*.

## Captured (rubric)

- **Hero:** ~25 words. "A nice place for JavaScript. Instantly deploy from Claude Code, or any agent." Emphasizes ease/accessibility + low activation energy ("Get started with a template").
- **Aliveness without scale (the lesson):** no "1M+ deployments" claims. Vitality instead via — **named testimonials** (6 voices from recognizable companies: Shopify, PixieBrix), a **template gallery** of concrete use cases ("Welcome New Stripe Subscribers"), activity nav ("Newest," "Trending"), and **20+ company logos** for network-effect perception. Trust = *who*, not *how many*.
- **Registry/community:** showcase = social proof disguised as utility; branded template cards with creator attribution (`stevekrouse/newStripeSubscriber`); Discord/GitHub links.
- **Sections (6):** hero → CTA + MCP link → templates → testimonials (~30% of viewport) → company logos → footer.
- **Visual/tone:** scrappy-polished hybrid; muted palette + single accent; clean-not-precious type; conversational copy ("A nice place," "like Zapier for developers"); real IDE screenshot; **anti-hype** ("avoids 'revolutionary'").
- **Nav:** dual-layer; assumes three modes — builders (Docs), browsers (Explore), deciders (Pricing/About). No single funnel.

## Lift for aDNA

- **Who-not-how-many** is the answer to aDNA's young-network problem: surface *named* operators/vaults/contributors + curated showcase templates, not raw counts. This is the explicit fix for the [[site_huggingface]] scale-mismatch.
- **Low-activation CTA + template gallery** → aDNA's "start from a vault template / fork the standard" entry.
- **Conversational, anti-hype tone** complements the Anthropic register at the warmer end.

## Avoid

- Inconsistent logo treatments / "organic growth" scrappiness can read unfinished; aDNA wants Val Town's *aliveness* with Linear's *craft discipline*.

## Install-forward / onboarding lens (2026-07-12)

Re-inspected for the P5 **R1 gate** (headless capture + WebFetch, 2026-07-12). **The homepage changed since June** — the hero is now "The fastest way to ship apps" / "developers and *vibe coders* ship apps and tools to live URLs," fronted by three product-UI screenshots (a Claude Code editor, a dashboard, a database). The install-forward read: **Val Town is the *sign-up-gated* counter-example** — there is **no install command anywhere** (no `curl`/`npm`/`deno`), the primary CTA is **"Sign Up,"** and getting-started is deferred behind auth; its demo-as-proof is **static screenshots**, not motion. **For aDNA this is an AVOID on the install axis:** aDNA's posture is *clone-and-run without an account* — the one-liner must be reachable and runnable **before** any signup (this is exactly why the M5.1a hero pill, **R2**, belongs front-and-centre). The *lift* stays what June found — who-not-how-many social proof — not the onboarding funnel. *(Currency note: the June "instantly deploy from Claude Code" framing is gone; exemplar homepages drift — re-inspect at cadence.)*
