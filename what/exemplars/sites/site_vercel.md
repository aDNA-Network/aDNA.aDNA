# site_vercel — Vercel

> **The dual-audience layering exemplar** — speaks to developers and decision-makers on the same page. Also a designated **AVOID** for marketing-adjective bloat (`writing-guidelines` §2): inspected to *lift the layering, refuse the fluff*.

## Captured (rubric)

- **Hero:** ~25 words. "Build and deploy on the AI Cloud" (dev) + "...developer tools and cloud infrastructure to build, scale, and secure a faster, more personalized web" (exec outcomes).
- **Sections (8+):** hero → case studies ("Runway: build times 7m → 40s") → use-case tabs (AI Apps / Web / Ecommerce / Marketing / Platforms) → dev layer (framework names, AI SDK snippet) → exec layer (enterprise security, ROI metrics) → AI Gateway (live model-popularity data) → customer logos → CTA.
- **Above-fold focus:** hero + a **case-study metric** (developers see the number; executives see the business category tabs).
- **Dual-audience layering:** dev signals (Next.js/Nuxt/Svelte, CLI/git, code sample) interleaved with exec signals (security, "95% reduction in load times"); performance claims bridge both.
- **Demo-as-design:** the AI Gateway shows **live usage data** (e.g., model popularity %) — credibility via transparency, not a static screenshot.
- **Density:** high but modular (each section self-contained); light/dark toggle.

## Lift for aDNA

- **Layer two readers on one page** — aDNA's "developer building a vault" + "org/partner evaluating the network" can coexist: technical proof (real config/registry) + value framing (what the democracy enables) in adjacent registers.
- **Live data as proof** (Vercel's usage %; aDNA's: live vault/registry counts, federation activity) beats static claims.
- **Specific metrics over adjectives** — "40s builds" not "blazingly fast."

## Avoid (explicit — this is an AVOID-list site)

- **Marketing-adjective bloat:** "fast, scalable, and reliable," "build, scale, and secure" — generic filler. The doctrine's AVOID table forbids these; restraint points come from *measured claims*, not adjectives.
- **8+ sections** is over the homepage budget; aDNA holds 5 ±1.
