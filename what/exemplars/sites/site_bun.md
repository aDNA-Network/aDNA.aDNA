# site_bun — Bun (bun.sh)

> **The canonical command-in-hero.** A CLI-installed open-source dev tool that makes the install one-liner *the* above-the-fold focus. The install-forward reference for aDNA's own clone-and-run posture — and evidence that a serious tool proves itself with a **command + a benchmark**, not a video.

## Captured (rubric)

- **Hero:** ~25 words — "Bun is a fast JavaScript all-in-one toolkit" + a one-line descriptor. The headline shares the fold with the install block.
- **Above-fold focus:** the **install command block** — the dominant element, not the words.
- **Install treatment (hero-level, copy button, dual-platform tabs):**
  - Linux & macOS: `curl -fsSL https://bun.sh/install | bash`
  - Windows: `powershell -c "irm bun.sh/install.ps1 | iex"`
  - Primary CTA **"Install Bun v1.3.14"**; secondary "View install script."
- **Demo-as-proof:** a **benchmark bar chart** ("Bundling 10,000 React components" — 269 ms vs esbuild/Rspack/…), **7 static syntax-highlighted code examples** (HTTP server, WebSocket, Postgres query…), and perf tables ("30× faster installs"). **No video, no GIF, no terminal cast** — proof is *copy-pasteable code + numbers*.
- **Social proof:** user logos (Lovable, CodeRabbit, Replit, Cursor), case studies (Claude Code, Railway, Midjourney), ~20 one-word developer tweets.
- **Visual:** dense (13 sections); monochrome black/beige/grey with a navy CTA; sans body + monospace code; entrance-only motion (auto-scroll testimonials).

## Lift for aDNA

- **Command-in-hero with a copy button** — the direct precedent for the M5.1a install pill (**R2**, shipped). Bun proves the pattern is the *dominant* fold element for a CLI tool, not a footnote.
- **Proof by artifact, not motion** — Bun's "demo" is a runnable code block + a benchmark. This is the strongest evidence for keeping **R1's demo lightweight and honest** (a real terminal, not a hero video). It also backs **R4**: keep code/ASCII where a terminal *is* the honest surface.
- **Dual-platform tabs** — the honest analog for aDNA's one-line prereq note (**R5**): macOS/Linux/Windows without download cards.

## Avoid

- **13 sections is well over the 5 ±1 homepage budget** ([[front_page_doctrine]] §2) — Bun is a mature multi-tool with more to sell; aDNA holds the budget.
- **Benchmark-swagger tone** ("30× faster") is a *speed* claim aDNA can't and shouldn't mimic — aDNA's proof is *an agent doing real work*, not a throughput number (the `writing-guidelines` AVOID register bars "blazingly fast").
