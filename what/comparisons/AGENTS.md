# what/comparisons/ — Agent Guide

## What's Here

aDNA vs. other knowledge architectures — honest, respectful positioning. Each file compares aDNA with one alternative system across multiple dimensions. The tone is not marketing: "These are good systems. Here's what aDNA adds — and what it costs."

One file per comparison.

## Working Rules

- **Naming**: `comparison_{adna_vs_system}.md` (underscores, lowercase)
- **Required frontmatter**: `type: comparison`, `created`, `updated`, `status`, `compared_to` (name of the other system), `categories` (list of comparison axes), `last_edited_by`, `tags`
- **Structure**: Overview of Both Systems → Comparison Table → Where aDNA Excels → Where the Other System Excels → When to Choose Which → Sources
- **Honest and fair**: Acknowledge where the other system is genuinely better. Never disparage.
- **Cite sources**: Link to the other system's documentation, not just your characterization of it
- **Cross-linking**: Link to relevant aDNA concepts that explain the differentiators
- **Set audit trail**: Update `last_edited_by` and `updated` on every edit

## Load/Skip Decision

**Load this directory when**:
- Answering "how is aDNA different from X?"
- Helping someone decide whether aDNA fits their needs
- Writing positioning or educational content

**Skip when**:
- Working within aDNA (comparisons are for external-facing context)
- Operational execution

**Token cost**: ~250 tokens (this AGENTS.md). Individual comparisons are ~2,000-3,000 tokens each.
